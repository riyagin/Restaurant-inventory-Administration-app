## Import Flow (two-phase, like POS import)

### Validation rules
- `full_name`, `join_date`, `position`, `branch`, `base_salary`, `working_days_per_month`, `effective_date` required.
- `position` and `branch` matched case-insensitively against existing names; unknown â†’ row error (do NOT auto-create).
- Duplicate `employee_code` (in file or DB) â†’ row error. Duplicate `full_name`+`dob` in DB â†’ warning (importable, flagged in preview).
- Amounts must be non-negative integers (rupiah). Dates must parse as `YYYY-MM-DD` or native Excel dates.
